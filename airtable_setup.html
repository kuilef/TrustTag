TrustTag Airtable Setup Guide

This guide will help you set up an Airtable base for use with the TrustTag Chrome extension.

Step 1: Create an Airtable Account

If you don't already have an Airtable account, go to airtable.com/signup to create one.

Step 2: Create a New Base

Once logged in:

  1. Click the "Add a base" button
  2. Select "Start from scratch"
  3. Name your base (e.g., "TrustTag Database")

Step 3: Set Up the Table Structure

Rename the default table to "Websites" and set up the following columns:

Column Name Type Description
address Single line text The website URL or pattern to match
short remark Single line text A brief warning message
Note text Long text A more detailed explanation
Source URL Link to source or further reading

Important: The column names must match exactly as shown above, including capitalization.

Step 4: Add Sample Data

Add some sample entries to test the extension:

address short remark Note text Source
example.com false information This site contains misleading information about various topics. https://factcheck.org/example
wikipedia.org/*Israel* biased content Some articles may contain biased information. https://example.com/analysis
fakenews.com known fake news site This site is known to publish fabricated stories. https://mediabias.org/fakenews

URL Pattern Tip: You can use wildcards (*) in the address field to match multiple pages. For example, wikipedia.org/*Israel* will match any Wikipedia page with "Israel" in the URL.

Step 5: Create a View

Create a Grid view that will be used by the extension:

  1. Click the "+" next to "Grid view" in the top-left
  2. Select "Grid view"
  3. Name it "TrustTag View"

Step 6: Get the Airtable URL with Base ID and Table ID

To get the correct Airtable URL for your table:

  1. Go to your Airtable workspace
  2. Open your base and select the table you want to use
  3. Look at the URL in your browser - it should look like https://airtable.com/appXXXXXXXXXXXXXX/tblYYYYYYYYYYYYYY/...
  4. Copy the URL up to and including the table ID (the part that starts with "tbl")

The URL should look like: https://airtable.com/appXXXXXXXXXXXXXX/tblYYYYYYYYYYYYYY

Tip: The extension will extract both the Base ID (starts with "app") and the Table ID (starts with "tbl") from this URL. You can also enter just the Base ID if you want to use the first table in your base.

Step 7: Get API Key (Required)

You need an API key to access Airtable data through the API:

  1. Go to your Airtable account page
  2. Under the API section, click "Generate API key"
  3. Copy the generated API key (it should start with "pat" followed by a long string)
  4. Enter this key in the TrustTag extension settings

Important: The API key is required for the extension to work. The extension uses the Airtable API to access your data, which requires authentication with an API key.

Next Steps

Now that your Airtable base is set up:

  1. Install the TrustTag Chrome extension
  2. Click the TrustTag icon in your browser toolbar
  3. Enter your Airtable URL and API key (if needed)
  4. Set your preferred sync interval
  5. Click "Save Settings"

The extension will now sync with your Airtable base and display warnings on matching websites.

Testing: You can use the included test.html file to test if your extension is working correctly with your Airtable data.
